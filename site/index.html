Consensus

DAG-BFT
consensus.

Blocks form a DAG, not a chain. Round-based validators produce signed vertices, BFT rules finalize.

Vertex Structure

Multi-Parent Blocks

Every vertex references all known tips, forming a directed acyclic graph. Multiple validators produce vertices concurrently in the same round without conflict.

Finality

Byzantine Finality

A vertex is finalized when 2/3+ of active validators have at least one descendant of it. Once finalized, it cannot be reverted. No 6-block wait.

Timing

Round-Based Timing

Configurable round intervals replace PoW as the rate limiter. Validators produce exactly one signed vertex per round. Clean async timing via tokio::interval.

Network

TCP Peer Network

Length-prefixed JSON over TCP. Peer discovery, transaction relay, DAG vertex propagation, and automatic orphan resolution with parent buffering.

Cryptography

Ed25519 Signatures

Real asymmetric cryptography via ed25519-dalek. Transactions and DAG vertices are Ed25519-signed. On-chain verification: blake3(pubkey) == address.

Architecture

Minimal Codebase

Three crates total: coin (crypto primitives + consensus), network (P2P), node (binary). No frameworks, no VMs, no smart contracts. Just a cryptocurrency.

Run a Node

Start validating
in 60 seconds.

One binary, no configuration files, automatic peer discovery.

Terminal 
# Build the node
cargo build --release -p ultradag-node

# Start validating (generates a keypair automatically)
cargo run --release -p ultradag-node -- --port 9333 --validate

# Connect a second validator to the first
cargo run --release -p ultradag-node -- --port 9334 --seed 127.0.0.1:9333 --validate

# Check chain status via HTTP RPC
curl http://127.0.0.1:10333/status

# Generate a new keypair
curl http://127.0.0.1:10333/keygen

# Send UDAG coins
curl -X POST http://127.0.0.1:10333/tx \
  -d '{"from_secret":"<sk_hex>","to":"<addr_hex>","amount":5000000000,"fee":100000}'
